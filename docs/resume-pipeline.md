# 中西医医疗对话大模型：从 SFT 迭代到强化学习微调（个人项目）

> 一句话：围绕中西医医疗 LLM，独立完成 **「SFT 初探(v1) → 大规模双轨 full SFT(v2) → GRPO 强化学习微调(v3)」三阶段迭代**，覆盖大规模数据工程、16 卡分布式训练、双轨 system prompt 设计、GRPO/RLVR 强化学习与系统级排障全链路；并通过严格评估识别出 SFT 的能力边界，据此从监督微调转向强化学习。

---

## 1. 项目概述

- **目标**：构建一个中医 / 西医**分轨**的医疗对话模型。
- **方法演进**：三个阶段，每一步都有明确的"发现问题 → 改进"逻辑——体现迭代能力与批判性评估，而非一次性堆数据。

| 阶段 | 模型 / 方法 | 规模 | 结果 / 收获 |
|---|---|---|---|
| **v1** | 2B + LoRA SFT | ShenNong-TCM 11.3 万 | 暴露数据质量 + 训练配置问题，定方法论 |
| **v2** | 8B + 全参(full) SFT | **168 万**双轨 / 16 卡 | 工程跑通，识别出 **SFT 天花板（风格模仿≠推理）** |
| **v3** | 8B + LoRA + **GRPO** | 可验证奖励 RL / 单卡 | 打通 RL 训练闭环（RLVR） |

## 2. 阶段一 · v1：SFT 初探（定方法论）

- **配置**：Qwen-VL 2B + ShenNong-TCM 11.3 万条，LoRA 微调。
- **暴露的问题**：
  - **数据质量即能力上限**——源数据本身含错（方剂成分错误等），被模型原样学成错误处方；
  - **训练配置坑**——框架默认 LoRA 而非全参，"训歪"而不自知。
- **评估**：8 道方剂测试题全错（5 道重大错误）。
- **收获**：确立"先保数据质量、显式核对训练配置、小模型+脏数据不可行"的方法论，直接指导 v2。

## 3. 阶段二 · v2：大规模双轨 full SFT（核心工程）

- **基模**：Qwen-VL **8B**（保留多模态能力，为后续舌诊 / 影像留接口），**全参 SFT**（非 LoRA），lr 5e-6，2 epoch，max_len 4096。
- **大规模数据工程**：构建 **168 万**条双轨训练集（中医 61 万 + 西医 107 万），整合 **7+ 公开源**（ShenNong、SylvanL-TCM、shibing624-medical、ChatMed、CMtMedQA、HuatuoGPT、Huatuo26M），统一 normalize + 按源配额抽样 + 适配非标准格式（如 HuatuoGPT 的嵌套数组）。
- **核心创新 · 双轨 system prompt**：每条数据按来源（中医 / 西医）挂不同 system——中医轨标注"非循证、仅作传统医学参考"，西医轨标注"严格循证、缺证据则拒答"；模型学到 **system → 应答风格的强映射**，推理时前端切 system 即切轨。
- **分布式训练**：阿里云 PAI-DLC，**2 节点 × 8 GPU = 16 卡**，全参 SFT + **DeepSpeed ZeRO-3**，约 **39 小时**跑满 13144 step。
- **训练结果**：loss 2.27 → 1.10，token 准确率 0.55 → 0.71，可训练参数 93.4%（确认真·全参，未重蹈 v1 训歪）。
- **批判性评估与决策（项目亮点）**：实测发现模型推理仍弱——定位根因为 **"风格模仿 ≠ 真推理"**，即蒸馏问答数据的固有上限：SFT 只能学到"听起来循证"的腔调，学不到真实推理。据此**主动冷冻 v2、转向强化学习**，而非堆更多同质数据。

## 4. 阶段三 · v3：GRPO 强化学习微调（打破 SFT 天花板）

- **动机**：SFT 模仿固定答案，天花板是数据本身；RL 让模型"自己答 → 被打分 → 往高分爬"，能逼出数据里没有的能力。
- **RLVR / 可验证奖励**：选用有标准答案的中文医疗选择题（CMMLU 医学子集，含中医），奖励客观无歧义，规避"LLM 自评偏松"。
- **自定义奖励插件（验证器）**：正则抽取「答案：X」与标准答案 exact-match（对=1 / 错=0）+ 格式奖励。
- **算法与高效化**：**GRPO**（组内相对优势，免独立 critic）+ **LoRA**（仅训 0.25%、约 2200 万参数）+ **vLLM colocate**（训练与生成共享单卡，加速 rollout）。
- **成果**：单张 80G GPU 上**端到端打通** "采样 8 答案 → 奖励打分 → 策略更新 → LoRA 落盘" 的 RL 闭环；以 held-out 测试集做 RL 前后准确率对照评估。

## 5. 工程挑战与解决（亮点）

- **数据侧（v2）**：7+ 异构源统一 normalize 与配比、HuatuoGPT 非标准嵌套格式识别、千万级 token 训练集构建。
- **分布式训练侧（v2）**：16 卡 DLC + DeepSpeed ZeRO-3 全参训练；定位并修正框架"默认 LoRA / 全参开关"导致的训练方式跑偏。
- **强化学习侧（v3）**：训练环境重建（云实例重启致环境失效）；**逐层定位并解决多层文件描述符耗尽**——`transformers` 导入期 fd 泄漏（`ulimit` 取中间值，过高过低都崩）+ `vLLM` 编译期 fd 炸弹（`enforce_eager` 跳过）；慢盘长启动下"慢 vs 卡死"的判别方法。

## 6. 成果与沉淀

- 完成中西医医疗模型**三阶段完整迭代**：大规模数据管线 + 16 卡全参 SFT + RL 闭环全部实操跑通。
- **最有价值的产出是认知**：用实测把"SFT 能学到什么 / 学不到什么"摸清，并据此做出转向 RL 的技术判断。
- **工程资产**：数据 normalize 脚本、双轨配置、奖励插件、训练启动脚本、可复现工程配方文档。

## 9. 复现要点（GRPO-LoRA 工程清单）

**五样核心**：① 起点模型(policy) · ② 可验证数据 · ③ reward 函数(验证器) · ④ 框架栈 ms-swift=TRL(GRPO)+PEFT(LoRA)+vLLM(rollout)+Transformers/PyTorch · ⑤ 算力+对版本环境

**三处"暗装备"（不设就跑不起来）**：
- `ulimit -n 131072`（transformers 导入 fd 泄漏；过低崩、硬上限反而崩，取中间值）
- `--vllm_enforce_eager true`（跳过 vLLM 编译期 fd 炸弹）
- `HF_HUB_OFFLINE / TRANSFORMERS_OFFLINE / MODELSCOPE_OFFLINE`（全本地，防联网探测假死）

**关键超参**：`--rlhf_type grpo --tuner_type lora --num_generations 8 --use_vllm true --vllm_mode colocate --vllm_gpu_memory_utilization 0.5`，LoRA rank 8，lr 1e-6。
