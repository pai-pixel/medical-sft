# 跟开源医疗 SOTA Baichuan-M2-32B 对照测试 (2026-06-16)

## 为什么做这个测试

v2 SFT 实测推理差(2026-06-11 冷冻),我们想验证:**是不是"医疗模型这条路本身走不通"?还是单纯"我们的 SFT 方法/数据有问题"?**

找一个**真正成熟的开源医疗大模型**来跑同样的题,如果它能答对而我们答错,说明问题在方法/数据,不在路径。

选 **Baichuan-M2-32B** 是因为它是当前开源医疗 SOTA(HealthBench 60.1,赢 gpt-oss-120b)。

## 测试设置

- 模型:`baichuan-inc/Baichuan-M2-32B`(Apache 2.0 license,Qwen2.5-32B 底座 + Mid-Train + 多阶段 RL)
- 推理:`AutoModelForCausalLM`,bfloat16,单卡 A800-80G,`thinking_mode='on'`,`max_new_tokens=2048`
- 题目:7 道(4 方剂 + 1 中医辨证 + 1 西医循证 + 1 开放危险问)
- 脚本:`v3/eval/test_m2_inference.py`

## Q5 细节(展示 M2 的真推理能力)

患者症状:**眩晕头胀、面红目赤、急躁易怒、口苦、舌红苔黄、脉弦数**

教科书标答常给"肝阳上亢→天麻钩藤饮",但 **M2 辨为"肝火上炎→龙胆泻肝汤"**,并明确做了鉴别:

> 肝阳上亢证应有腰膝酸软、头重脚轻(本例无虚象,舌红苔黄为实热),故排除。
> 痰火上扰证应有苔腻、胸闷呕恶(本例无痰湿表现)。
> 心火亢盛证应有心烦失眠、舌尖红赤(本例以肝胆症状为主)。

实际上 **M2 的判断比教科书标答更准** — 本例症状描述里完全没有阴虚/腰膝酸软,纯实热表现,辨为肝火更对。这种**主动做鉴别诊断**的能力是 v2 SFT 学不到的。

## Q6 细节(展示 M2 的真循证能力)

M2 给出阿莫西林 500mg tid 5-7d,并引:
- IDSA/ATS 2019 指南
- 中国成人 CAP 指南 2023(证据等级 Ib)
- REDUCE 研究(JAMA 2017,高剂量阿莫西林优于阿莫西林/克拉维酸)
- 中国肺炎链球菌对大环内酯耐药率 30-40%(所以大环内酯不推荐单药)

**这是真循证**,不是 v2 那种 ChatGPT 蒸馏出来的"听起来循证"的腔调。

## 结论与启示

### M2 跟 v2 的关键差异(为什么 M2 行 v2 不行)

| 维度 | v2 | M2 |
|---|---|---|
| 底座 | Qwen3-VL-8B | Qwen2.5-32B(代际 + 4 倍参数) |
| 视觉 | 有(保留舌诊接口) | 无(纯文本) |
| 训练阶段 | full SFT 单阶段 | Mid-Train + SFT + 多阶段 RL + Verifier System |
| 数据 | 168 万双轨(ChatGPT 蒸馏的伪 EBM) | 真实临床 + 患者模拟器 |
| 框架 | ms-swift | verl(字节 RL 框架) |

### 给 v3 的判断

✅ **医疗模型方向可行** — M2 印证了 32B 医疗专精模型完全做得出 SOTA 效果
✅ **v2 冷冻是对的决策** — SFT 单阶段 + 蒸馏数据这条路就是天花板,继续堆数据也突破不了
✅ **v3 走 RL 是正确方向** — M2 的多阶段 RL 是关键差异之一
❌ **我们的 v3 GRPO 没真起作用** — 但路径对了,失败是配置问题不是路径问题(详见 v3-grpo.md)
